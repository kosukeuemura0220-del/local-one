# Local ONE

Local ONE は、Mac、Windows、iPhone、Android、ブラウザ/PWA 間で、ファイル、動画、画像、GIF、フォルダ、`.env`、テキスト、秘密情報をローカル優先で送受信するアプリです。

デスクトップ側でローカルハブを起動し、信頼済みのWeb UIを公開します。Mac/Windowsではデスクトップアプリとして使え、iPhone/Androidはブラウザ/PWAからQRペアリングで接続します。

## できること

- Mac / Windows デスクトップアプリ対応
- iPhone / Android ブラウザ・PWA対応
- `http://<デスクトップのIP>:5174` から同一Wi-Fi内で接続
- 期限付きQR・6桁コードペアリング
- 信頼済みデバイスのセッション管理
- ファイル、動画、画像、GIF、アーカイブ、フォルダ、`.env` のドラッグ&ドロップ送信
- アプリ側のファイルサイズ上限なし
- 大容量ファイル向けチャンクアップロード
- 受信ハブ側に再開用チャンクメタデータを保持
- アップロード後のSHA-256検証
- テキストメモ転送
- キーや`.env`向けの秘密モード
- 秘密テキストは履歴でマスク表示し、`state.json`には保存しない
- ダウンロード、コピー、受信フォルダへの保存
- デスクトップモードで受信フォルダを変更可能
- ネットワークが許可する場合、Local ONEハブ同士をmDNSで検出

## ローカル起動

```bash
npm install
npm run dev
```

Web UIを開きます。

- 同じMac上のブラウザ: `http://127.0.0.1:5174`
- iPhone/Android: アプリに表示されるLAN URLを同じWi-Fi内で開く

デスクトップシェルで起動する場合:

```bash
npm run dev:desktop
```

Web UIをビルドする場合:

```bash
npm run build
```

ビルド後に本番用ローカルハブを起動する場合:

```bash
npm start
```

## Mac / Windows ビルド

```bash
npm run dist:mac
npm run dist:win
```

ビルド成果物は `release/` に出力されます。

## ペアリング手順

1. MacまたはWindowsでLocal ONEを起動します。
2. 右側の `デバイスをペアリング` を押します。
3. iPhone/Androidでは、カメラでQRコードを読み取るか、ペアリングリンクを開きます。
4. カメラのないWindowsなどでは、QR下に表示される6桁コードをロック画面に入力します。
5. デバイス名を確認してペアリングします。
6. 接続した端末に信頼済みセッションが保存されます。

ペアリング用のQR・6桁コードは5分で期限切れになります。ローカル端末以外のデバイスは、転送APIを使う前にペアリングが必要です。

## 大容量ファイル

Local ONEはアプリ側のファイルサイズ上限を設けません。大容量転送は標準で8 MiBのチャンクに分割します。実際の上限は次の条件に依存します。

- 送信元/受信先の空き容量
- ブラウザのメモリやバックグラウンド実行の制約
- Wi-Fi速度と安定性
- OSのスリープ設定
- モバイルブラウザのアップロード制約

複数GBの転送では、両方の端末を同じLAN上で起動したままにしてください。

## 秘密モード

秘密テキストはメモリ上に保持され、キューではマスク表示され、設定されたTTL後に期限切れになります。`state.json`には意図的に書き込みません。

`.env`などの秘密ファイルは通常のファイル転送として扱います。履歴上は秘密として印が付きますが、配置できるように受信ファイルとして保存されます。使い終わったら転送を削除してください。

## データ保存先

開発時の標準保存先:

```text
./data
```

Electronビルドではアプリのユーザーデータ領域を使います。環境変数で上書きできます。

```bash
LOCAL_ONE_DATA_DIR=/path/to/data LOCAL_ONE_INBOX_DIR=/path/to/inbox npm start
```

## 検証

```bash
npm run build
npm test
npm audit
```

`npm audit` は脆弱性0件になる想定です。

## セキュリティメモ

Local ONEは信頼できるローカルネットワーク向けです。インターネットへ公開するファイルサーバーではありません。

- `47110` ポートを公開インターネットに出さないでください。
- QRペアリングは信頼できる端末だけに使ってください。
- 秘密ファイルを配置した後は、必要に応じて転送履歴を削除してください。
- 機密性の高い転送や大容量転送では、プライベートWi-Fiを使ってください。
- iOS/Androidのネイティブ共有シート対応は、同じAPIの上に後から追加できます。
